# Compilers

本 README 文件为 `compilers` 目录提供说明。

## 简介

此目录包含与编译器构造相关的源代码文件，主要运用 `lex` (Flex) 和 `yacc` (Bison) 工具。这些文件是学习和实践编译原理，特别是词法分析和语法分析的示例。

## 文件说明

目录中的文件根据其扩展名可分为两类：

*   **`.l` 文件**: 这些是 `lex` 或 `flex` 的源文件，用于定义词法分析器。词法分析器负责读取源代码的字符流，并将其转换为一系列的“词法单元”（tokens）。
*   **`.y` 文件**: 这些是 `yacc` 或 `bison` 的源文件，用于定义语法分析器。语法分析器接收词法分析器生成的词法单元，并根据预定义的语法规则构建一个抽象语法树（AST）。

## 如何使用

要编译和运行这些示例，您需要在您的系统上安装 `flex` 和 `bison`。

### 编译 `.l` 和 `.y` 文件

通常，编译过程如下：

1.  **处理 `.l` 文件**：使用 `flex` 将 `.l` 文件编译成 C 源代码。
    ```bash
    flex <filename>.l
    ```
    这将生成一个名为 `lex.yy.c` 的文件。

2.  **处理 `.y` 文件**：使用 `bison` 将 `.y` 文件编译成 C 源代码。
    ```bash
    bison -d <filename>.y
    ```
    这将生成 `y.tab.c` 和 `y.tab.h` 两个文件。

3.  **链接生成 C 代码**：使用 C 编译器（如 `gcc`）将生成的 C 文件链接成一个可执行文件。
    ```bash
    gcc lex.yy.c y.tab.c -o my_parser
    ```

### 运行

编译完成后，您可以运行生成的可执行文件来解析输入。

```bash
./my_parser < input.txt
```

请注意，具体的编译命令可能因文件的不同而有所差异。

## 开发

要修改或扩展这些示例，您可以：

*   **修改 `.l` 文件**：添加或修改正则表达式，以识别新的词法单元。
*   **修改 `.y` 文件**：扩展语法规则，以支持新的语言结构。

在修改后，请重新执行编译步骤以生成新的解析器。
